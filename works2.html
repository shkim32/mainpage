WeatherMap

개발환경 및 소개

오픈 API를 이용하여 날씨정보와 지도 정보를 불러와 활용하여 한국의 날씨를 확인할 수 있는 사이트를 구현하였습니다. MySQL을 이용해 한국어로 시/군/구 의 부분 검색만으로도 날씨를 검색할 수 있도록 데이터를 구축하여 검색기능을 구현하고, 현재 위치와 대표 도시들의 경우 버튼을 활용하여 검색할 수 있도록 하였습니다.


WeatherMap 프로젝트는 제가 API를 공부하기 위해 처음으로 혼자서 공부하고 구현하여 만들었습니다. 이전 게시판을 만들었을때에는 오로지 PHP만을 이용하여 작업하였는데, 위 프로젝트를 하며 Javascript와 jQuery 기능들을 넣어가며 해당 스킬들을 서로 필요한 때에 부르기 위해 호출하는 방법에 대해 공부해야 했습니다. 또한, 개발 과정에서 더 넣고싶은 기능과 빼고싶었던 기능들이 생겨나 계속해서 계획을 변경하였었는데, 이 과정을 통해 프로젝트에 있어 처음 일의 진행 이전에 넣고싶은 기능들은 구상한 후 기획하는 것이 중요하다는 것을 깨달았습니다.

수행업무

  1. - 사이트 입장시 Openweathermap, 카카오맵, Geolocation API 를 사용하여 현재위치를 불러와 해당 위치의 날씨 표시
  2. - 한국어로 부분 검색할 수 있도록 도시들의 한글파일과 위도 경도를 불러올 수 있도록 데이터 구축
  3. - 한국의 대표 도시들의 버튼을 만들어 데이터 연동하여 검색할 수 있는 기능 생성

트러블 슈팅


- 문제배경

도시 검색을 구현 후 도시 버튼을 만들었을 때, 코드의 충돌로 두가지 기능 모두 이용되지 않고 계속해서 현재위치만 뜨는 오류


-해결방법

두가지 기능 코드 중 겹치는 부분을 찾아내어 if문을 이용하여 검색 여부에 따른 기능 구현을 만들었습니다.
버튼 클릭시 위도.경도를 아이디 값으로 불러 검색하고, 아이디 값이 없을 때 검색기능 혹은 검색 내용이 없을 때는 현재위치를 불러오도록 재구현 하였습니다.


-해당경험을 통해 알게된 점

처음으로 여러가지 언어들을 함께 이용하여 페이지를 만들다 보니 구현하고 싶은 항목이 생겨 추가할 때마다 PHP에서 Javascript를 호출하거나, Javascript에서 PHP를 호출해야하는 과정을 겪었습니다. 여러가지 방법과 시도를 통해 페이지를 구현하며 새로운 기술을 알아가고 공부하게 된 것 같습니다.

사용기술

  1. - Javascript, jQuery
  2. - PHP
  3. - MySQL
  4. - API = Openweathermap, 카카오맵, Geolocation
